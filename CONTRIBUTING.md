# Contributing to SOGo Community of Practice

Thank you for contributing! This repo is a shared knowledge base for SOGo practitioners.

## How to Contribute

1. **Fork** this repository
2. **Create a branch** for your changes (`docs/deployment-ldap`)
3. **Add or edit** documents under the appropriate `docs/` subdirectory
4. **Submit a Pull Request** with a clear description of what you're adding

## Guidelines

- Use **Markdown** for all documents
- Keep language **clear and concise**
- Include **environment context** (OS, SOGo version, scale) when sharing configs
- Redact **sensitive information** (passwords, IPs, internal hostnames)
- Follow the existing structure — if in doubt, open an issue first

## Review Process

PRs are reviewed by CoP maintainers. We aim for a 7-day review window.
